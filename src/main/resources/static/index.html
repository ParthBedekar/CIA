// pipeline

Five steps from commit to clarity

01

Clone & Diff

We clone your repo and diff the last two commits to get a precise list of changed lines — nothing more, nothing less.

↓
02

Parse AST

Changed files are fed into a Java + JavaScript AST parser. We extract method additions, removals, import changes, and class-level modifications.

↓
03

Build Graph

Every CodeUnit becomes a node. Every dependency becomes an edge. The graph lives in Neo4j and is rebuilt fresh per analysis run.

↓
04

Trace Impact

Starting from every changed node, we walk the graph backwards — finding everything that calls, imports, or inherits from the changed unit.

↓
05

Map to Processes

Affected files are grouped by their top-level directory (your "process"). You see not just which files are affected, but which business domains.

// about

Built for teams who move fast

Rippler started as an internal tool to answer one question: "If I change this service, what else breaks?" Code reviews catch bugs but not blast radius. Test suites catch regressions but don't tell you where to look.

Rippler fills that gap. Give it a repo, it gives you a map.
